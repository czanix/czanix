## 🚀 Boilerplates — Clone e Comece a Codar

Padrões que usamos em produção há 15 anos. Open source. Clone, rode `docker compose up`, e comece.

### Backend APIs

| Stack | Repo | Destaque |
|-------|------|----------|
| **Node.js / TypeScript** | [`boilerplate-api`](https://github.com/czanix/boilerplate-api) | PostgreSQL, Express, Clean Architecture, OWASP |
| **Python / FastAPI** | [`boilerplate-api-python`](https://github.com/czanix/boilerplate-api-python) | AsyncIO, Result Pattern, Pydantic |
| **C# / .NET 8** | [`boilerplate-api-csharp`](https://github.com/czanix/boilerplate-api-csharp) | MediatR, CQRS, Entity Framework |
| **Java / Spring Boot** | [`boilerplate-api-java`](https://github.com/czanix/boilerplate-api-java) | Hexagonal, Flyway, Sealed Interfaces |
| **Go / Gin** | [`boilerplate-api-go`](https://github.com/czanix/boilerplate-api-go) | pgx, zero-alloc, Makefile |
| **Node.js / MongoDB** | [`boilerplate-api-nodejs`](https://github.com/czanix/boilerplate-api-nodejs) | Express, text search, indexes |
| **Rust / Axum** | [`boilerplate-api-rust`](https://github.com/czanix/boilerplate-api-rust) | SQLx compile-time, Tokio, zero GC |

### Frontend

| Stack | Repo | Destaque |
|-------|------|----------|

| **Vue 3** | [`boilerplate-frontend-vue`](https://github.com/czanix/boilerplate-frontend-vue) | Composition API, Pinia, Router |

### Mobile & IA

| Stack | Repo | Destaque |
|-------|------|----------|
| **Flutter** | [`boilerplate-flutter`](https://github.com/czanix/boilerplate-flutter) | Riverpod, offline-first, GoRouter |
| **AI / RAG Pipeline** | [`boilerplate-ai-rag`](https://github.com/czanix/boilerplate-ai-rag) | Vertex AI, pgvector, multi-provider |

### O que todos compartilham

- ✅ **Clean Architecture** — domínio puro, infraestrutura desacoplada
- ✅ **Result Pattern** — sem exceção para fluxo de negócio
- ✅ **OWASP Security Headers** — segurança não é opcional
- ✅ **Docker** — `docker compose up` e tudo funciona
- ✅ **BIGINT PK + UUID público** — performance + segurança
- ✅ **Testes** — unit + integration com exemplos reais
- ✅ **Documentação** — README com "quando NÃO usar" (trade-offs)
